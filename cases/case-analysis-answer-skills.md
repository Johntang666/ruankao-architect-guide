# 系统架构设计师案例分析答题技巧与模板

> 更新时间：2026-04-02

## 一、先把边界说清楚

- 案例分析是系统架构设计师三科之一，当前常见机考口径下和综合知识连考，总时长 240 分钟。
- 近年的平台总结普遍将案例分析描述为 `5` 道主观大题，作答 `3` 道，通常是“第 `1` 题必做，后面 `4` 选 `2`”。
- 这科本质上不是背定义，而是根据场景做架构判断、分析问题、给出方案，并把理由写清楚。

## 二、先记住评分导向：阅卷老师在找“得分点”

案例分析没有论文那种长篇展开空间，阅卷时更看重下面几件事：

1. 你有没有真正回答题目，而不是写一堆相关但不对题的概念。
2. 你有没有把答案拆成清晰的得分点，而不是写成一大段散文。
3. 你有没有把“为什么”写出来，也就是方案和场景、质量属性、约束条件之间的关系。
4. 你有没有用专业术语把逻辑说清楚，而不是泛泛说“提高性能”“增强安全”。

可以把它压缩成一个作答原则：

> 先给结论，再分点展开；每一点都要能对应题干中的某个关键词。

## 三、通用作答套路：先审题，再拆题，再落点

这是最值得反复练熟的一套通用流程。

### 1. 审题四件套

拿到题以后，不要立刻写答案，先在草稿区快速标出四类信息：

- 业务场景：系统是什么，服务谁，核心目标是什么。
- 动作词：题目到底让你“分析”“比较”“说明”“设计”“补图”“找问题”还是“提改进”。
- 约束条件：性能、安全、可靠性、成本、兼容性、并发量、数据规模。
- 输出形式：是文字说明、图形补全、表格比较，还是混合型问题。

可以直接套这句起手式：

> 本题关注的核心是 `场景/对象` 下的 `问题类型`，约束条件包括 `A/B/C`，因此答题要围绕 `维度1/维度2/维度3` 展开。

### 2. 拆题三步法

- 先判断本题属于哪类题型，不要把“架构比较题”按“问题诊断题”去写。
- 再把大问拆成小问，逐一列出答点。
- 最后决定每个小问的结构，不同小问不要混写。

### 3. 落笔结构

最稳的是下面这个顺序：

1. 先用一句话写结论。
2. 再按 `1. 2. 3.` 分点展开。
3. 每一点都写“机制/原因/效果”。
4. 最后用一句话回扣题目或给出推荐方案。

## 四、案例分析高频题型的回答模板

下面这些模板不是“万能标准答案”，而是为了让你在考场上稳定拿基础分和中档分。

### 1. 架构风格/技术方案比较题

常见问法：

- 比较两种架构风格的优缺点
- 判断某种架构是否适合当前系统
- 在多个技术方案里做选型并说明理由

推荐模板：

1. 先写结论：本题更适合采用 `方案A`，或 `A/B` 分别适用于不同目标。
2. 写核心机制：该方案的核心结构、协作方式、控制流/数据流方式是什么。
3. 写优势：围绕性能、扩展性、维护性、复用性、解耦性展开。
4. 写局限：哪些场景下不适合，会带来什么额外复杂度。
5. 写适配理由：结合题目给出的业务特点、并发量、部署环境、数据流形式，说明为什么它适合或不适合。

答题句式：

- `该方案的核心机制是……，因此在……方面更有优势。`
- `结合题干中的……约束，采用该方案可以……，但需要注意……问题。`
- `如果系统更强调……，则更适合……；如果更强调……，则更适合……。`

### 2. 质量属性/性能/可靠性/安全类题

常见问法：

- 分析系统的性能瓶颈
- 从质量属性场景角度评价设计
- 给出高可用、可靠性、安全性改进措施

推荐模板：

1. 明确目标：本题核心质量属性是性能/可用性/安全性/可维护性。
2. 写影响因素：瓶颈或风险来自哪里，是计算、存储、网络、并发控制、单点故障还是权限边界。
3. 写设计策略：缓存、异步化、冗余、限流、隔离、审计、熔断、降级、主从/集群、备份恢复等。
4. 写权衡：引入这些措施后会增加什么成本或复杂度。
5. 写预期效果：能缓解什么问题，提升什么指标。

答题句式：

- `从性能角度看，当前瓶颈主要出现在……，原因是……。`
- `可通过……策略降低……带来的风险。`
- `该策略能够提升……，但同时需要在……方面做额外控制。`

### 3. 找问题/提改进/优化建议题

常见问法：

- 当前架构存在哪些问题
- 应如何优化或改造
- 某设计会带来哪些风险

推荐模板：

直接按下面五段写最稳：

1. 问题：先点出现象或缺陷。
2. 原因：为什么会这样。
3. 风险：如果不改会造成什么后果。
4. 改进：给出具体措施，不要只写原则。
5. 收益：改完之后改善什么。

答题句式：

- `当前存在……问题，根因在于……。`
- `这会导致……风险，例如……。`
- `建议采用……进行改进，具体做法包括……。`
- `改进后可在……方面获得提升。`

这类题最怕两种错误：

- 只写“应该优化数据库、加强缓存”这种空话。
- 只写改进措施，不写为什么改、解决什么问题。

### 4. UML/ER/DFD/建模补全题

常见问法：

- 补用例图、类图、顺序图、状态图
- 画 E-R 图或补实体关系
- 根据业务流程补数据流图

推荐模板：

1. 先找主体：参与者、实体、处理、数据存储、边界对象。
2. 再找关系：谁和谁交互，关系类型是什么，多重性怎样。
3. 再找约束：主键、外键、包含/扩展、同步/异步、状态迁移条件。
4. 最后用一句业务规则解释你为什么这样补。

作答要点：

- 不要只盯图形本身，先把业务规则说清楚。
- 如果是 E-R 图，优先检查主键、联系、联系的基数。
- 如果是顺序图，优先检查对象、消息方向、先后顺序。
- 如果是用例图，优先检查参与者、系统边界、包含/扩展关系。

### 5. 数据库/缓存/中间件/分布式机制题

常见问法：

- 解释 Redis、消息队列、分布式锁、事务、复制、路由等机制
- 结合业务场景说明为什么选某个中间件
- 分析一致性、可用性、性能、解耦之间的取舍

推荐模板：

1. 先交代场景需求：为什么需要缓存、消息、锁、事务协调。
2. 再写核心机制：这个中间件或机制如何工作。
3. 再写收益：它解决了什么问题。
4. 再写风险：会引入什么副作用。
5. 最后写适用前提：什么条件下才适合这样用。

答题句式：

- `引入……的直接目的是解决……问题。`
- `其核心机制是……，因此能够在……方面带来收益。`
- `但如果……处理不当，可能导致……，因此还需要配合……措施。`

### 6. 新技术场景题

常见问法：

- AI、知识图谱、区块链、云原生、嵌入式智能如何落进具体系统
- 某新技术在当前业务里的作用、优势和风险

推荐模板：

1. 不先空谈概念，先回到业务痛点。
2. 明确这项技术在系统中的位置，是核心业务能力、支撑能力还是辅助能力。
3. 写落地方式：接入点、数据流、依赖组件、训练/推理/存储/治理链路。
4. 写收益：准确率、效率、自动化、扩展性、智能化。
5. 写风险：成本、复杂度、可解释性、安全、运维门槛。

这类题的核心不是“你懂不懂热词”，而是“你能不能把技术装回系统”。

## 五、考场上的时间和选题策略

这部分不是官方硬性规则，而是基于当前考试形式和真题作答节奏给出的实战建议。

### 1. 先做题目选择，不要平均用力

- 第一题通常必做，先看清问法。
- 后面选做题优先挑自己熟悉的题型，不要看到新技术就慌，也不要盲目追难题。
- 有图题、建模题、数据库题基础好的，通常更容易稳定拿分。

### 2. 单题内部时间分配

可按下面的节奏控制：

- `2-3` 分钟审题和标关键词
- `1-2` 分钟列答题框架
- 主体作答阶段优先写稳能拿分的点
- 最后 `1` 分钟检查是否漏问

### 3. 作答顺序

- 先把确定会写的点写出来，再补展开。
- 不要在单个难点上卡太久。
- 如果小问很多，优先确保每个小问都有内容，不要顾着把前两问写得很满，后面空着。

## 六、平时训练时怎么把模板练成反应

建议不要只按年份刷题，而是按“题型”建训练集：

- 架构风格与方案比较
- 质量属性与架构评估
- UML/ER/DFD 建模
- 数据库与中间件机制
- 高并发、可靠性、安全
- AI/区块链/知识图谱/云原生等新技术场景

每做完一题，至少复盘三件事：

1. 这题属于什么题型。
2. 我有没有按模板把得分点写完整。
3. 我丢分是因为不会，还是因为会但没写到点上。

更有效的训练方式不是“做一遍对答案”，而是：

- 第一遍按考试时间写
- 第二遍按标准模板重写
- 第三遍把这题沉淀到自己的题型模板里

## 八、压缩成一页的案例分析速记总结

如果临考前只允许看一页，我建议只记下面这些：

### 1. 通用作答主线

- 先审题，再作答；先看场景、动作词、约束条件、输出形式。
- 先给结论，再分点展开；每一点都尽量对应题干关键词。
- 每个答点最好都包含 `原因/机制 + 方案/判断 + 效果/风险`。
- 不写大段散文，优先让阅卷老师一眼看到得分点。

### 2. 高频题型速记

- 架构比较题：`结论 -> 机制 -> 优点 -> 局限 -> 适用场景`
- 质量属性题：`目标 -> 风险/瓶颈 -> 策略 -> 权衡 -> 指标`
- 找问题提改进题：`问题 -> 原因 -> 风险 -> 改进 -> 收益`
- UML/ER/DFD 题：`找主体 -> 找关系 -> 找约束 -> 用业务规则校验`
- 中间件机制题：`场景需求 -> 核心机制 -> 收益 -> 风险 -> 适用前提`
- 新技术场景题：`业务痛点 -> 技术位置 -> 落地方式 -> 收益 -> 风险`

### 3. 考场避免失分的三个动作

- 每个小问都先列关键词，再下笔。
- 不会的题也先写能确定的术语和结构，不要整题空着。
- 最后回看一遍是否漏问，特别是“解释原因”“说明理由”“比较优缺点”这类动作词。

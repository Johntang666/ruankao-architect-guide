## 两个系统各自的技术亮点速查

### 紫金资金管理系统（主系统）

1. qiankun 微前端：14 子应用、独立构建、统一接入、跨应用状态
2. 统一交易网关：`/cmsmessage + prcscd` 处理码分发
3. 交易生命周期：RG → AU → CK → AC → AK 状态机
4. 公共能力下沉：web-frame（登录/HTTP/字典）、web-components（工作流/记账/打印）
5. 外部系统集成：ESB 对接 CFETS/中债/核心/名单平台/风险暴露
6. 报表中心：163 路由、动态表头、打印中心
7. Koa/Electron 本地承载：内网桌面化部署

### FiboRule 风控引擎（附加系统）

1. 规则外置：以业务变化率为边界拆分系统
2. 四中心架构：指标中心 + 策略中心 + 引擎中心 + 监控中心
3. 8 种策略类型：规则、脚本、评分卡、决策表、决策树、名单、模型、RPC
4. 可视化决策流：节点编排、递归执行、并行/聚合/分组/冠军挑战
5. 版本与快照：每次发布保存配置快照，支持审计追溯
6. 管理端与执行器分离：读写分离、独立扩容、安全隔离
7. Canal 同步：MySQL 变更自动同步 Redis 缓存

## 论文核心叙事线

无论考哪个题目，论文的核心叙事线都是：

```
业务矛盾 → 架构决策 → 技术落地 → 权衡分析 → 效果反思
```

对于"资金管理系统 + FiboRule"组合，核心叙事线是：

```
交易主流程稳定 vs 风控规则频繁变化
  → 规则外置为独立平台
    → 指标中心 + 策略中心 + 引擎中心 + 监控中心
      → 版本快照 + 审计追溯
        → 规则变更效率提升 + 交易系统稳定性保障
```

这个叙事线可以适配几乎所有的论文题目，只需要在"技术落地"环节切换到题目要求的具体技术即可。
